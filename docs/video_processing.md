# Пайплайн обработки видео

Когда пользователь отправляет боту ссылку на видео с YouTube (или файл), происходит серия ресурсоемких преобразований, управляемых в основном модулем `handlers/processing.py`.

## Основные этапы

1. **Загрузка (Download)**
   Модуль `download.py` использует `yt-dlp` для извлечения исходного видео в оптимальном качестве и сохранения его в локальной директории.
   
2. **Получение транскрипции (Transcription & LLM)**
   - Аудиодорожка извлекается из видео и подаётся в `transcribe.py`.
   - Используется модель **OpenAI Whisper** (чаще всего работает на CPU или GPU, при первом запуске подгружаются веса). Транскрибация возвращает временные метки (таймкоды) произнесённого текста.
   
3. **Анализ моментов и нарезка (Scenes / Moments)**
   - `scenes.py` применяет `PySceneDetect` для автоматического разделения видео на смысловые куски.
   - `moments.py` проводит дополнительный анализ, выбирая самые динамичные/важные части (в т.ч. с использованием LLM).
   
4. **Удаление тишины (Auto-edit)**
   С помощью инструмента `auto-editor` в модуле `autoedit.py` из фрагментов вырезаются продолжительные промежутки тишины, что делает видео-формат быстрым и ориентированным на формат Shorts/Reels/TikTok.

5. **Вертикальное кадрирование (Vertical Crop & Scale)**
   Модуль `vertical.py` преобразует горизонтальное видео (16:9) в вертикальное (9:16). В проекте есть продвинутый алгоритм \`facecam_top_split\`:
   - Определяется зона лица/вебкамеры (модели YOLO).
   - Лицо компонуется в верхней трети экрана.
   - Основной контент (например, геймплей) размещается в нижней части экрана.

6. **Создание и рендеринг субтитров (Subtitles)**
   С помощью `subtitles.py` создаются ASS (.ass формат) файлы, которые стилизуются под крупные, динамичные субтитры в стиле популярных Shorts. Эти субтитры "запекаются" в видео с помощью FFmpeg.

7. **Финальная сборка и публикация/сохранение**
   Итоговое видео возвращается пользователю в Telegram-бот, либо опционально выгружается в облако (через интеграцию Mail.ru Cloud). Также с видео отдаются сгенерированные хештеги (`tagging.py`).
